Create a Pen

To create a Pen you must have a codepen account and you must be logged in.

Select the Create Pen link to open the Editor view and start writing the code to create a Pen. The standard editors are HTML, CSS and JavaScript. As you write in the editors the result is shown in the preview area.

The free version of codepen allows only one project but an unlimited number of Pens. For this reason the freeCodeCamp curriculum suggests the use of pens to do the projects for their Responsive Web Design certification. It should be noted that the freeCodeCamp example projects are Pens, and the freeCodeCamp test suite is in a Pen that users can fork.

HTML Editor

This editor requires only the code that would go between the body tags. For anything outside the body element click the gear icon in the upper left. As an alternate, click the Settings button and choose HTML from the Pen Settings column.

The HTML Settings article has information on personalizing this editor.

CSS Editor

This is the same as an external stylesheet. All CSS selectors are coded here producing the CSS Ruleset for the Pen.

The CSS Settings article has information on personalizing this editor.

The JS Settings article has information on personalizing this editor.

Saving Changes

Saving a new Pen or saving changes to an existing Pen is as easy as clicking the Save button after starting the creation of a new Pen or when editing an existing one.

It should be noted that when first creating a new Pen or working on an existing one, Autosave is off by default. Once the Save button is clicked, Autosave is enabled and will stay enabled until the Pen is closed. Autosave is on a timer. This means that not every keystroke is saved instantly. The timing trigger kicks off to save your work so data loss is not common but, it is possible that some loss may occur.

Even with Autosave on user's can, and should, use the Save button to save their work.

Unsaved Work

Codepen will give a warning:

  • when attempting to leave a Pen using the browser buttons
  • when attempting to close the browser window / tab
  • when attempting to refresh a Pen that has unsaved work

If the user leaves or refreshes the page anyway, changes will not be saved.

Pen Editor Settings

HTML Settings

Click the gear icon in the upper left of the Editor to

  • select an HTML Preprocessor such as;
    • None (default)
    • Haml
    • Markdown
    • Slim
    • Pug
  • Add Class(es) to <HTML>
    • In CodePen, whatever you write in the HTML editor is what goes within the <body> tags in a basic HTML5 template. You don't have access to higher-up elements like the <html> tag. If you want to add classes there that can affect the whole document, this is the place to do it.
  • Access the Stuff for <head> box to add head elements such as
    • meta tags, for example: <meta name="viewport" content="width=device-width, initial-scale=1">
    • A title for the document
    • A link to a font, for example: <link href="https://fonts.googleapis.com/css2?family=Roboto&display=swap" rel="stylesheet">

CSS Settings

Click the gear icon in the upper left of the Editor to

  • select a CSS Preprocessor such as
    • None (default)
    • Less
    • SCSS
    • Sass
    • Stylus
    • PostCSS
  • select a CSS Base using either
    • Normalize
    • Reset
    • Neither
  • set Vendor Prefixing using either
    • Autoprefixer
    • Prefixfree
    • Neither
  • Add External Stylesheets / Pens
    • Any URL's added here will be added as links in order, and before the CSS in the editor. If you link to another Pen, it will include the CSS from that Pen. If the preprocessor matches, it will attempt to combine them before processing.

JS Settings

Click the gear icon in the upper left of the Editor to set

  • a JavaScript Preprocessor such as
    • None (default)
    • Babel
    • TypeScript
    • CoffeeScript
    • LiveScript
  • Add External Scripts / Pens
    • Any URL's added here will be added as <script>s in order, and run before the JavaScript in the editor. You can use the URL of any other Pen and it will include the JavaScript from that Pen.
  • Add Packages
    • Search for and use JavaScript packages from npm here. By selecting a package, an import statement will be added to the top of the JavaScript editor for this package.

Additional Settings

Click the Settings button to access additional settings such as

  • Pen Details - Give a Pen Title, a Pen Description and add Tags
  • Privacy - PRO only
  • Behavior - Settings to Save Automatically, Auto-Update the Preview, Format on Save
  • Editor - Indent code, Code Indent width
  • Template - Make a Template or select Template URL
  • Screenshot - PRO only

Pen Views Explained

  • Editor view is the most important page on CodePen. Here using the standard editors, HTML, CSS and JavaScript is where users create, or edit, a Pen.
  • Details view shows a large preview of the Pen. The code can be turned on to view but not edit. This view contains more social information about the Pen such as;
    • Who liked it
    • Any comments that were made
    • Sharing buttons
    • The License
    • and more
  • Full Page view is a way to look at, or share, a Pen that uses almost the whole screen to show the preview. This view does not show code.
  • Debug mode is for looking at the Pen full-screen with no iframe. The view opens in a new tab and is only for you, the logged user so it is not great for sharing but it is great for debugging.

The following are available to PRO (paid) accounts and are mentioned, without elaboration, in this document.

  • Live view PRO users only
  • Collab Mode and Professor Mode are views available to PRO users and are used for working in real time with other people.
  • Presentation mode is another PRO user view used for viewing the Pen Editor on an overhead projector.
